# 🥾 В Поход! — цифровой кооперативный прототип

Цифровая кооперативная игра-путешествие по мотивам настольной игры бренда
**«Походы Первых. Больше, чем путешествие»** (АНО «Больше, чем путешествие»).
Прототип для Форума «Шерегеш».

Команда из **2–6 игроков** вместе идёт по процедурной карте из плиток местности:
**База → Промежуточная цель → Главная цель**. Победа командная, проигрыша нет —
в финале команда получает **1–3 звезды** и автоматическую **«историю похода»** для соцсетей.

Платформа — **браузерная веб-игра** (React + TypeScript + Vite). Лёгкая графика
(эмодзи + CSS, ~112 КБ gzip) — работает на медленном интернете. Режим —
**поочерёдный (hotseat)** на одном устройстве. Сборка статическая → бесплатный хостинг
(GitHub Pages / Netlify / любой статический хостинг). Легко обернуть в Telegram Mini App.

## Запуск

```bash
cd vpohod
npm install          # при NODE_ENV=production добавьте: NODE_ENV=development npm install --include=dev
npm run dev          # дев-сервер на http://localhost:5173
# или продакшн-сборка:
npm run build        # результат в dist/ — заливается на любой статический хостинг
npm run build:single # один автономный файл dist-single/index.html (offline, скинуть судье)
npm run preview
```

> Быстрый старт для показа: открой `?demo` (напр. `http://localhost:5173/?demo`) —
> сразу запускается демо-партия на 4 игроков со случайной картой.

### Дизайн
Спокойная премиальная палитра (глина / шалфей / янтарь), типографика
**Fraunces** (заголовки) + **Manrope** (текст), единый набор **кастомных SVG-иконок**
(`src/ui/icons.tsx`) вместо разнородных эмодзи, мягкие тени, микро-анимации
(бросок кубика, открытие плиток, появление пешек, звёзды финала).

## Что реализовано (по приоритизации ТЗ)

### ✅ Обязательно (ядро, критерий №1)
- **Процедурная карта** из 5 типов местности (Равнина, Лес, Водоём, Болото, Горы),
  3 размера (4×4 / 5×5 / 6×6, ≤36 плиток), сложные плитки ближе к центру,
  **гарантированная проходимость** База → Промежуточная → Главная.
- **6 ролей** по схеме «1 постоянное + 1 особое действие»: Разведчик, Исследователь,
  Хранитель лагеря, Хранитель атмосферы, Хранитель впечатлений, Спасатель.
- **Ход-цикл с ≤2 бросками**: бросок погоды на раунд + бросок шага игрока.
- **Маршрут** База → Промежуточная цель → Главная цель (без возврата).
- **Командное взаимодействие**: обмен артефактами (минимум) + см. ниже.
- **Финал со звёздами** (1–3) + **«история похода»** с кнопкой «Поделиться».
- **Поочерёдный режим** на 2–6 игроков.
- **Фирменный стиль** «Больше, чем путешествие» (палитра, тёплый тон).

### ➕ Желательно
- **6 погодных карт** (значки, штрафы только −1/−2), одна общая на раунд.
- **4 артефакта** (Компас, Плавсредство, Головной убор, Спецснаряжение) —
  часть в старте (Спасатель), часть — находки на карте.
- **Командные карты** («Второе дыхание», «Привал», «Сплочённость»).
- **Совместная задача**: доставить артефакт на Промежуточную цель → награда команде.
- **Жетоны истории** (Хранитель впечатлений) влияют на финальный рейтинг.

### ✨ Опционально (реализовано сверх плана)
- **Единое меню** со сквозным потоком: меню → настройка → игра → передача хода → события → финал → меню.
- **«Поход дня»** — одна и та же карта для всех в течение дня (сидируемый генератор) + локальная таблица результатов.
- **Значки-достижения** (10 шт.) с прогрессом в профиле и показом новых на финале.
- **Случайные события на тропе** (родник, уставший путник, короткая тропа и др.) с выбором — добавляют «доброту» и разнообразие.
- **Передача устройства** (hotseat) — экран «Передайте устройство игроку X» между ходами (отключается).
- **Профиль игрока** (localStorage): статистика, значки, результаты дня.
- **Связка с реальными походами** (миссия БЧП) — см. ниже.
- **Шеринг** «истории похода» (`navigator.share` / копирование).
- **Сборка в один файл** (`npm run build:single`) — автономный offline `index.html`.

## Связка с реальными походами (миссия БЧП)

Игра подталкивает ребёнка к настоящему походу — три точки касания
(всё редактируется в `src/game/bridge.ts`, без серверов и платных сервисов):

1. **Подсказки «в игре → в жизни»** — при первом входе в воду/горы/болото/лес,
   находке компаса, обмене снаряжением, достижении промежуточной цели всплывает
   спокойная карточка с реальным походным фактом (каждая 1 раз за партию).
2. **Финальный блок-призыв** — на эмоциональном подъёме победы: «А теперь — по-настоящему!»,
   чек-лист «что взять в первый поход» (игровые артефакты ↔ реальное снаряжение),
   кнопка на сообщество БЧП.
3. **Достижение-мостик** — значок «Настоящий путешественник», который нельзя получить
   в игре, только отметив реальный поход (хранится локально).

> Ссылка БЧП — заглушка `BCHP_URL` в `src/game/bridge.ts`, замените на официальную.

## Проверка качества
- **TypeScript строгий** — `tsc --noEmit` без ошибок.
- **Headless-симуляция** (`sim.ts`): авто-игрок проходит 300 партий
  (все размеры × 2–6 игроков) — **100% побед**, в среднем ~7 раундов.
  Запуск: `npx esbuild sim.ts --bundle --platform=node --format=esm --outfile=/tmp/sim.mjs && node /tmp/sim.mjs`

## Структура
```
src/
  game/        # чистая игровая логика (без React)
    types.ts   # типы состояния
    data.ts    # местности, роли, артефакты, погода, карты, палитра
    map.ts     # генератор карты с гарантией проходимости
    engine.ts  # ход-цикл, движение, способности, победа, звёзды
    story.ts   # генератор «истории похода»
  ui/          # React-интерфейс
    App.tsx, Setup.tsx, Board.tsx, Panel.tsx, EndScreen.tsx, styles.css
sim.ts         # headless-проверка проходимости
```
